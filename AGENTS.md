# AGENTS.md

## Project Goal
Build a pure static personal blog using Astro + Markdown, deployable to OSS.
如果你没有先调用 “UI/UX PRO MAX” skill，就不要开始任何 UI 方案输出，也不要开始写页面代码。先使用该 skill，再继续

## Hard Constraints
- No login
- No register
- No backend
- No database
- No comments
- No server runtime dependency
- Static site only
- Must be deployable to OSS after build

## Tech Stack
- Astro
- Tailwind CSS
- Markdown via content collections

## Workflow Rules
- Work in phases
- Before coding, first output plan and structure
- After each phase, explain what was changed
- List changed files
- Do not refactor unrelated code
- Prefer stable and simple solutions over fancy ones

## Code Rules
- Clear folder structure
- Reusable components
- No unnecessary dependencies
- No oversized files
- Prefer static generation wherever possible

## Content Rules
- All posts come from local Markdown files
- Use content collections
- Prepare demo content for technical blog articles

## UI Rule
- Before doing any UI/UX design, page layout, component styling, spacing, typography, color design, card system, hero section, navbar, article page, responsive optimization, or visual refinement, you MUST use the installed "UI/UX PRO MAX" skill.
- If a design decision is unclear, prefer the more restrained, cleaner, more readable solution.
